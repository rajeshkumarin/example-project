AngularJS Model

The ng-model directive binds the value of HTML controls (input, select, textarea) to application data.

The ng-model Directive

With the ng-model directive you can bind the value of an input field to a variable created in AngularJS..

<!DOCTYPE html>
<html>
<script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.4.8/angular.min.js"></script>
<body>

<div ng-app="myApp" ng-controller="myCtrl">
Name: <input ng-model="computer">
</div>

<script>
var app = angular.module('myApp', []);
app.controller('myCtrl', function($scope) {
    $scope.computer = "zoome computer";
});
</script>

<p>Use the ng-model directive to bind the value of the input field to a property made in the controller.</p>

</body>
</html>

Two-Way Binding

The binding goes both ways. If the user changes the value inside the input field, the AngularJS property will also change its value:

Example four text fields are bound together with four ng-model directives:

<!DOCTYPE html>
<html>
<script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.4.8/angular.min.js"></script>
<body>

<div ng-app="myApp" ng-controller="myCtrl">
Name:<input ng-model="name">
<p><b>This is great contory:</b>-<span ng-bind="name"></span></p>
</div>
<script>
var app = angular.module('myApp',[]);
app.controller('myCtrl', function($scope) { $scope.name = "I love india";});
</script>

<p>Use the ng-model directive to bind the value of the input field to a property made in the controller.</p>

</body>
</html>

Validate User Input

The ng-model directive can provide type validation for application data (number, e-mail, required):

The span will be displayed only if the expression in the ng-show attribute returns true.

<!DOCTYPE html>
<html>
<script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.4.8/angular.min.js"></script>
<body>

<form ng-app="" name="myForm">
    Email:
    <input type="email" name="myAddress" ng-model="text">
    <span ng-show="myForm.myAddress.$error.email">Not a valid e-mail address</span>
</form>

<p>Enter your e-mail address in the input field. AngularJS will display an errormessage if the address is not an e-mail.</p>

</body>
</html>

Application Status

The ng-model directive can provide status for application data (invalid, dirty, touched, error):

<!DOCTYPE html>
<html>
<script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.4.8/angular.min.js"></script>
<body>

<form ng-app="" name="myForm" ng-init="myText = 'rahulsinha.kumar@gmail.com'">

Email:
<input type="email" name="myAddress" ng-model="myText" required>
<p>Edit the e-mail address, and try to change the status.</p>
<h1 style="color:red">Status</h1>
<p>Valid: {{myForm.myAddress.$valid}}(dineshwarpaswan3@gmail.com).</p>
<p>Dirty: {{myForm.myAddress.$dirty}}</p>
<p>Touched: {{myForm.myAddress.$touched}}</p>
<p>empty: {{myForm.myAddress.$empty}}(This is no data in box)</p>
<p>untouched: {{myForm.myAddress.$untouched}}</p>
<p>invalid: {{myForm.myAddress.$invalid}}(DineshwarPaswan@gmail.com)</p>
<p>ng-pending: {{myForm.myAddress.$ng-pending}}</p>
</form>

</body>
</html>
